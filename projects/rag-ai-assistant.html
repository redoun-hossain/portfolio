Project Overview

This project demonstrates a complete AI-powered Retrieval-Augmented Generation (RAG) workflow developed entirely with n8n. Instead of relying solely on an LLM, the assistant retrieves verified information from company documents stored in Google Drive, searches them through Pinecone Vector Database, and generates highly accurate responses using OpenAI GPT-4o. The workflow also includes confidence checking, human escalation, logging, and automation, making it suitable for real-world business deployment.

Complete Workflow

The workflow is fully built inside n8n. Documents are collected from Google Drive, converted into embeddings, stored in Pinecone, searched using vector similarity, and finally answered by GPT-4o before sending the response back through Telegram.

System Architecture

The architecture follows a Retrieval-Augmented Generation pipeline. User queries first pass through Telegram, then n8n orchestrates every step, retrieves relevant vectors from Pinecone, generates grounded answers using GPT-4o, and returns reliable responses.

Challenge & Solution

❗

The Challenge

The company received hundreds of repetitive support questions every day. Support agents spent hours searching documentation before replying. Customers often waited several hours for answers, especially outside office hours. This increased workload, reduced customer satisfaction, and slowed business operations.

✅

The Solution

A complete RAG AI Assistant was built with n8n. Documents are automatically indexed from Google Drive, embedded using OpenAI, stored inside Pinecone, and searched in real time. GPT-4o generates accurate responses based only on retrieved company knowledge. If confidence is low, the workflow escalates the conversation to a human agent automatically.
